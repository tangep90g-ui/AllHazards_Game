# 《全災變局：前線指揮官》 (AllHazards: Frontline Commander) 
**遊戲設計與架構基礎文件**

## 遊戲核心概念
**遊戲類型：** 2D 模擬經營 (Simulation & Idle) + 策略放置 (Base-Building) + 限時微遊戲 (Time-Management)
**核心訴求：** 透過這款結合《異塵餘生：庇護所》(Fallout Shelter) 與時間管理操作機制的遊戲，達到向玩家推廣「全災害管理 (All-Hazards Management)」四大階段的教育目的。

## 開發策略 (Hooks First)
為了首先抓住玩家的注意力並展現核心亮點，開發順序將採用 **Hooks First** 策略，從最高壓、最刺激的 **「應變 (Response)」** 階段著手，接著才拓展到整備、復原與減災等經營層面，讓玩家自然體會到前期整備的重要性。

---

## 遊戲四大階段機制

### 1. 應變 (Response) ——「黃金救援 180 秒」
* **階段定位：** 高頻率的「檢傷分類與派遣」微型遊戲。
* **核心玩法：** 在限時內（180秒）點擊與拖曳，處理現場亂象與傷患。
* **重點機制：** 
  * 玩家必須面對多線操作壓力：檢傷分類、指派處置、處理二次災害（如餘震導致路徑封閉）。
  * 結合真實的 START 檢傷分類標準（紅、黃、綠、黑），點擊傷患顯示生理數據來進行判斷。正確標記加分，錯誤扣分。

### 2. 整備 (Preparedness) ——「最強裝備構築 (Loadout)」
* **階段定位：** 出勤前的策略配置（類 Roguelite 的戰前準備）。
* **核心玩法：** 資源分配、設備配置與人員調度。
* **重點機制：** 
  * 收集與養成要素：升級救護車、招募擁有不同專長（高級救護員、搜救專家）的隊員。
  * 根據預警情報決定攜帶的機具或物資。若未裝備關鍵物品（如：生理監視器），在應變階段救護時將失去詳細數據，增加救援難度。

### 3. 復原 (Recovery) ——「重建與心理韌性」
* **階段定位：** 災後的資源重新分配與人員修復。
* **核心玩法：** 敘事選擇 (Narrative Choices)、資源抉擇。
* **重點機制：** 
  * 災後資源分配：先修復基礎設施還是重點安撫災民？
  * 人員壓力管理：前線人員具有「壓力值 (Sanity)」，若連續出勤則會崩潰，需透過心理復原機制進行恢復。

### 4. 減災 (Mitigation) ——「上帝視角的都市規劃」
* **階段定位：** 經營模擬 (Simulation)，長期放置與升級。
* **核心玩法：** 2D 網格建造系統與放置升級 (Idle)。
* **重點機制：** 
  * 建立城市防護中心，逐步投資於防震錨點、治水工程、公眾衛教。
  * 其投資能對「應變」階段產生直接的正面助益（例如：減災做得好，下次相同災害的輕傷比例增加、障礙物減少）。

## 技術堆疊方向
* **語言與框架：** JavaScript (ES6+), HTML5 Canvas 搭配 CSS 動畫 / React (待定)。
* **架構限制：** 嚴格遵守各模組的單一職責，狀態、遊戲迴圈 (`src/core/`)、遊戲階段邏輯 (`src/phases/`)、實體 (`src/entities/`) 與 UI 渲染 (`src/ui/`) 完全分離。
* **資料驅動：** 所有醫療邏輯與災害數據需獨立抽取為 JSON 配置檔置於 `assets/data/` 中。
